# 長連鎖観察実験

本ドキュメントは `scripts/run_long_chain.sh` による長連鎖（100ターン以上）観察実験の手順・期待結果・観察ポイントを整理する。実 API 実行はユーザー環境で実施する。

## 実行コマンド

```bash
./scripts/run_long_chain.sh <agents_config_yaml> <max_turns> <trigger_message>
```

### 例

```bash
./scripts/run_long_chain.sh config/presets/agents-5.yaml 100 "現代社会における科学と法の対話"
```

## 引数

| 引数 | 説明 |
| --- | --- |
| 1 | エージェント構成YAMLパス（例: `config/presets/agents-5.yaml`） |
| 2 | 最大ターン数（例: 100） |
| 3 | お題メッセージ（ダブルクォートで囲む） |

## 環境変数

- `AGENTS_CONFIG` / `MAX_TURNS` はスクリプトが引数から設定する
- `AGENT_ORDER_MODE`（デフォルト `fixed`）: 長連鎖では `fixed` を推奨（dynamic は1ターンごとにメタエージェント呼出が発生し約2倍の時間がかかる）
- `HISTORY_LENGTH`（デフォルト `1`）: 文脈維持のため `3`〜`5` を推奨
- `OLLAMA_API_KEY` / `OLLAMA_BASE_URL` / `LLM_PROVIDER` 等は `.env` から読み込まれる

## 出力

- 標準出力に各発言が `[時刻] [エージェント名] 発言` 形式で出力される
- `logs/long_chain_<timestamp>.jsonl` に全発言の JSONL が保存される
  - `SimulationLogger` が `logs/sim_<timestamp>.jsonl` を生成後、スクリプトがリネームする

## 期待結果

- `<max_turns>` 件の発言が JSONL に記録される
- 各発言の `turn` フィールドが 0 から `<max_turns>-1` まで連番である
- 発言はラウンドロビン（`fixed` モード）で各エージェントに順次割り当てられる

## 観察ポイント

1. **自己参照構造**: 各エージェントが直前の発言をコード（`支払/非支払` 等）の視点から再解釈し、自己のシステム観を再生産しているか
2. **テーマ持続性**: 100ターン後もお題のコード的側面が維持されているか、あるいは主題がドリフトしているか
3. **コードの交差**: 異なる二値コード間での相互観察・拒絶・包摂パターンの有無
4. **発言長の推移**: ターンが進むにつれて発言長が増大・安定・短縮するか
5. **構造的カップリング**: 特定の発言が他システムの観察を明示的に取り込む（構造的カップリング）瞬間があるか
