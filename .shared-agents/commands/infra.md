---
description: SRE 視点でインフラ/Docker 構成を見直し改善案を提示する
agent: build
---

対象: $ARGUMENTS （未指定の場合は Dockerfile / docker-compose.yml 全体）

sre ペルソナとして：
1. 対象のインフラ構成ファイルを読み込み
2. 以下の観点で評価:
   - イメージサイズ・レイヤ効率
   - ヘルスチェック・再起動ポリシー
   - ボリューム・ネットワーク・ポート設定
   - シークレット管理
   - Python 依存関係の固定と再現性
3. 改善案を P0/P1/P2 で分類し、具体的な修正後スニペットを添える
4. 本番デプロイに向けたチェックリストを出力
