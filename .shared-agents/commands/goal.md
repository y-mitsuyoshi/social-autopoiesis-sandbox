ユーザーが入力した目標: $ARGUMENTS

この目標を達成するため、以下のフェーズを順に実行する自律ループを開始してください。
各フェーズの完了時に `todowrite` で進捗を更新し、ユーザーに現状を報告した上で次フェーズに進むこと。

## フェーズ構成

1. **PRD 策定** — prd-manager ペルソナで PRD を作成し `docs/prd/<slug>.md` に出力
2. **Tech Spec 作成** — architect ペルソナで Tech Spec を作成し `docs/spec/<slug>.md` に出力
3. **アーキテクチャレビュー** — architect-reviewer ペルソナで Tech Spec をレビュー。P0 がある場合は フェーズ2 に戻る
4. **実装** — implementer ペルソナでコードを実装。自己修正ループを必ず実行
5. **チームレビュー** — team-reviewer ペルソナでパネルレビュー。P0 がある場合は フェーズ4 に戻る
6. **QA** — qa-engineer ペルソナでテストを拡充し検証。失敗時は フェーズ4 に戻る
7. **最終報告** — tech-lead ペルソナで結果を統合し、マージ可能か判定

## 制約
- 各フェーズの最大自動再試行は 3 回まで。3 回で解決しない場合はユーザーに状況を報告して一時停止
- 本番デプロイは絶対に自動で行わない
- すべての成果物は `docs/` 配下に整理し、コミットはユーザーの明示指示時のみ

## 開始
まず目標を分解し、`todowrite` で7フェーズを登録してから フェーズ1 を開始してください。
