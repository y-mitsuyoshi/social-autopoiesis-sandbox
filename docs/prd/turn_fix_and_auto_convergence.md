# PRD: TURN 1起算・ターンピッタリ終了・自動収束モード・ちかちか完全撤廃

## 概要
ユーザーからの要請：
1. **発話中のちかちか点滅の完全廃止**: アニメーション点滅を一切廃止し、人間が見やすい静的な洗練された枠線・バッジ表示に変更。
2. **TURN 1からの表示（1-indexed）**: 画面上のターン表示を 0 からではなく 1 から開始（`Turn 1`, `Turn 2`, ...）。
3. **MAX TURNS ピッタリ終了の修正**: 15ターン設定時に15回ぴったりで発話を終了し、表示上のターン数と実際のカウントを完全に一致させる。
4. **無制限・自動収束モードの追加**: MAX TURNS = 0 の際、議論が深まり意見がまとまった段階でモデレーター/システムが自動検知して議論を美しく収束・終了させるモードを追加。
5. **完了後の分かりやすい総括解説**: 議論が終了した後にルーマン理論に基づく分かりやすい解説を提示。

## 詳細要件

### 1. ちかちか点滅アニメーションの完全撤廃
- `HumanAvatar.tsx`, `RoundtableStage.tsx`, `App.tsx`, `RealtimeCyberMetrics.tsx`, `AutopoiesisGraphPanel.tsx` など全コンポーネントから、点滅（`animate-pulse`, `animate-ping`, `animate-bounce` 等）を削除。
- 発話中状態は「静的なアンバーゴールドの枠線＆ソフトな光芒シャドウ」、思考中は「静的な落ち着いた琥珀色バッジ」に統一。

### 2. Turn 1 起算化 (1-indexed Turn Display)
- 対話タイムライン、ラウンドテーブル、ログ表示、分析パネル等すべての画面表示において、`turn` 表示を `m.turn + 1` (1, 2, 3, ...) に統一。

### 3. MAX TURNS の正確な カウント・終了制御
- バックエンド `simulation.py` の `turn < max_turns` ループ処理および `agent_order` 順序を検証し、設定した回数（例: 15回）ぴったりでシミュレーションを終了するように修正。

### 4. 自動収束モード (MAX TURNS = 0)
- `max_turns == 0` (または画面で「自動収束モード」選択時) に、対話が十分に回って合意・収束に達したと判定（例: モデレーター/AI判定または作動的閉鎖スコアが閾値到達）された時点で「議論が収束しました」として自動終了。

### 5. 総括解説の自動表示
- 自動収束またはターン上限終了時に、会話全体のまとめ解説を分かりやすくUI上に提示。
