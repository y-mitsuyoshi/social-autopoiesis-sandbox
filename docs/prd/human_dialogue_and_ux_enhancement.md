# PRD: 人間らしい対話化・完了演出・ルーマン考察＆ガイド強化・UIチカチカ改善

## 概要
ユーザーからの改善要求：
1. **AIの対話スタイルの人間化**: マークダウン論文風の出力（箇条書き・表）を抑え、役割・二元コードを保ちながら人間らしく会話・議論するトーンにする。
2. **ターン上限到達時の完了状態と演出の明確化**: 完了時に発話者の点滅/speaking状態が残る問題を修正し、「シミュレーション完了」を明確に提示。
3. **シミュレーション完了後のオートポイエーシス総合考察**: 対話ログ全体の自己再生・構造的結合・作動的閉鎖をルーマン理論に基づいて深掘り考察するセクションを追加。
4. **ルマン先生解説パネルのボリューム強化**: `LuhmannTeacherPanel.tsx` の理論解説、Q&A、概念辞典を大幅に充実化。
5. **チカチカ点滅UI/UXの廃止と見やすさ向上**: `animate-ping` / `animate-pulse` / 点滅エフェクトを廃止・抑制し、落ち着いたエレガントな発光とハイライト表現に変更。

## 要求事項

### 1. 人間らしい対話スタイルのAIプロンプト改善
- バックエンド (`simulation.py`) およびフロントエンドのプリセット (`presets.ts`, 各 `yaml` ファイル) のシステムプロンプト・ガイドラインを更新。
- 「論文・報告書形式（表、箇条書きヘッダー、過度な見出し）を禁止し、人間同士が対話・口論・議論する口調で自然に返答すること」を指示。

### 2. 完了状態の厳密管理と完了通知UI
- `App.tsx` において `status === "completed"` または `status === "failed"` の場合、すべてのエージェントの `state` を即座に `idle` へリセット。
- シミュレーション終了時に目立つ完了バナー（例: 「🎉 シミュレーション完了 // 全ターンの作動的閉鎖が成立しました」）を表示。

### 3. ルーマンのオートポイエーシス総合考察
- シミュレーション完了後に対話全体の「作動的閉鎖の度合い」「システム間の刺激と応答」「二元コードの衝突と発展」を深く解説・分析する考察パネル/セクションを追加。

### 4. ルマン先生ガイドのコンテンツ大幅拡充
- `LuhmannTeacherPanel.tsx` の「社会システム論の解説」「初心者Q&A」「用語事典」をボリューミーかつ理解しやすく拡充。

### 5. UIの点滅（チカチカ）抑制
- `GlitchText` や `animate-ping`, `animate-pulse` などの過剰な高速点滅アニメーションを削除・静的アクセント/穏やかなカラーハイライトに変更。
