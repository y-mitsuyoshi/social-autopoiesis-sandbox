# PRD: ルーマン自己組織化社会システムシミュレーション ダッシュボード機能拡張

## 概要と目標
本製品要求仕様書（PRD）は、ニクラス・ルーマンの社会システム論を可視化する『自己組織化社会システム・サンドボックス』ダッシュボードへの追加機能および機能改善要件を定義する。

本機能拡張の主な目的は以下の通りである。
1. **LLM接続のタイムアウトおよびエラーへの耐性向上**: 不安定なLLM API通信環境においてもシミュレーションの強制終了を防ぎ、安定して長期のシミュレーション稼働を継続可能にする。
2. **オートポイエーシス証明ステータスおよび初心者向け解説パネルの追加**: 学術的で難解なシステム論の概念（作動的閉鎖など）が成立しているかを判定・可視化し、初心者にも直感的に学習できる環境を提供する。
3. **攻殻機動隊スタイルのサイバーアバター演出**: UIの近未来感を強化するため、幾何学模様アバターをネオン調の人間シルエットに変更し、発話中の電脳通信風ウェーブエフェクトを追加する。
4. **人間らしい自然な対話表現とターン平準化**: エージェント発言の偏りを防止する厳格なターン制御を導入し、AI同士の会話の脈絡や自然さを向上させる。

## ターゲットユーザー / ユースケース
- **U-1 社会システム論・社会学の学習者 / 学生**
  - ルーマン理論における「自己再生産」「作動的閉鎖」「二値コード」の概念が、実際のシミュレーション中にどのように作用し証明されるかを、ダッシュボード上で視覚的に学びたい。
- **U-2 近未来風UI/UXを好むデモ閲覧者**
  - 攻殻機動隊の「電脳空間」を思わせるスタイリッシュな画面で、エージェントたちの生き生きとした会話アニメーションを観察したい。
- **U-3 シミュレーション管理者 / 開発者**
  - LLMのタイムアウトや通信エラーによりシミュレーションが中断されるリスクを最小限に抑え、信頼性の高い長期実行環境を構築したい。

## 機能要件 (必須)

### 1. LLM タイムアウト & エラー耐性の向上 (LLM Timeout & Error Resilience)
- **FR-1.1**: バックエンド設定（`AppConfig`）で、環境変数 `LLM_TIMEOUT`（デフォルト値: `120.0` 秒）を正しくロード・適用可能にする。
- **FR-1.2**: `llm_client.py` 内の `OpenAICompatibleClient` および `GeminiClient` の `httpx.AsyncClient` 生成時に、上記 `LLM_TIMEOUT` をタイムアウト設定として適用する。
- **FR-1.3**: `llm_client.py` と `simulation.py` において、規定回数（最大3回）の接続リトライを行ってもタイムアウトやその他の通信例外（`LLMError`）が解消しない場合のフォールバック処理を実装する。
  - シミュレーションを例外で強制終了（クラッシュ）させず、ダッシュボードおよびログへフォールバックメッセージ（例: `「（環境からのノイズにより一時的に通信が途絶しています。システムは作動的閉鎖を維持しています）」` 等）を自動生成して出力する。
  - シミュレーションは次のターンの処理へと安全に進むこと。

### 2. オートポイエーシス証明ステータス & 初心者向け解説パネル (Autopoiesis Proof Status & Beginner-Friendly Panels)
- **FR-2.1**: シミュレーション実行中、**「作動的閉鎖（Operational Closure）」**が達成されたかどうかをプログラムでリアルタイム判定するロジックを実装する。判定条件は以下のすべてを満たす状態とする。
  1. **全非メタエージェントの参加**: シミュレーションに参加しているすべての非メタ（non-meta）エージェント（機能システム）が少なくとも1回以上発言していること。
  2. **自己参照的連鎖の形成**: 発言連鎖のネットワークグラフにおいて、各システムが他のシステムから応答を受け、かつ他のシステムへ応答を返す自己参照的なループ（循環参照）が形成されていること。
  3. **二値コードの活性化**: 各システムの二値コード（例：経済の「支払／非支払」、科学の「真／偽」など）の双方が発言を通じて活性化、あるいは適切に記録されていること。
- **FR-2.2**: フロントエンドのダッシュボード上の目立つ位置に、オートポイエーシス証明ステータス（Autopoiesis Proof Status）インジケータを表示する。
  - 作動的閉鎖が成立した場合: **`"AUTOPOIESIS PROVEN"`** （サイバーグリーン/ネオンカラーで強調表示）
  - 作動的閉鎖が未成立の場合: **`"UNPROVEN / OPERATIONAL CLOSURE INCOMPLETE"`** （タクティカルイエロー/警告色で表示）
- **FR-2.3**: 初心者向けの解説パネル（SocietyPanel または EducationalPanel のアップデート）を配置し、ルーマンの核心概念である「自己再生産」「作動的閉鎖」「二値コード」「構造的カップリング」を平易な日本語で解説する。また、上記「証明ステータス」の達成条件についても初心者向けに分かりやすく記述すること。

### 3. 攻殻機動隊スタイル サイバーアバター (Ghost-in-the-Shell Style Cybernetic Avatars)
- **FR-3.1**: `AgentAvatar.tsx` およびダッシュボード上に描画されている従来の幾何学模様アイコンを、ネオン調に美しく発光する人間のシルエット（ベクタープロファイル・顔パーツを含まないスタイリッシュな輪郭線）を模したSVG表現へ置き換える。
- **FR-3.2**: エージェントが `"speaking"`（発話中）状態に遷移した際、アバターの周囲にアニメーションする音波/周波数ライン（電脳通信風ウェーブエフェクト）を描画し、通信が行われていることを動的に示す。

### 4. 人間らしい対話改善 & バックエンドターン制御 (Human-like Dialogue & Scheduler Turn Enforcement)
- **FR-4.1**: バックエンドの `run_simulation`（`backend/app/simulation.py`）を修正し、シミュレーションのサイクル内において、**すべての非メタエージェントが必ず均等に1回ずつ発言すること**を保証するスケジューラを導入する（「全員が1回発言を終えるまで、同じエージェントが複数回連続で選ばれない」厳格なターン制御）。
  - `fixed` モード：設定された順序通り全非メタエージェントが一巡する。
  - `dynamic` モード：メタエージェント（モデレータ）に提示する次の発言者の選択肢候補（candidates）から、そのサイクル内で「既に発言したエージェント」を動的に除外する。全員が発言し終えた時点で候補リストをリセットし、次のサイクルを開始する。
- **FR-4.2**: 各エージェントに送信されるシステムプロンプトやコンテキスト構造を改良し、AIエージェントの発言が単なる独立した学術主張の独り言ではなく、前の発言者の内容を踏まえ、対立・同意・疑問などを交えた人間味のある「自然な対話（ディスカッション）」として展開されるようにプロンプトエンジニアリングを最適化する。

## 非機能要件 (パフォーマンス、UX等)
- **NFR-1: UI描画パフォーマンスの最適化**
  - アバターの周波数ウェーブおよびネオンエフェクトのアニメーションにおいて、CSSや `framer-motion` の過剰な再計算によるブラウザの描画遅延やCPU負荷を防ぎ、スムーズな60fps描写を維持する。
- **NFR-2: 堅牢なエラーロギング**
  - LLM呼び出しのタイムアウトやエラーによるフォールバック発生時、バックエンドログにおいて「通信障害によるフォールバック発言の生成」であることが明確に記録・識別できるようにする。
- **NFR-3: 後方互換性の維持**
  - 既存のYAML形式のプリセットファイル（エージェント設定など）やAPI構造の大部分を破壊することなく、ターン制御ロジックを統合する。

## 受け入れ基準 (Acceptance Criteria)
- [ ] **LLMエラー耐性の検証**: `LLM_TIMEOUT`（デフォルト `120.0`）が環境変数経由で設定され、LLM API呼び出しに適用されること。また、擬似的な接続タイムアウトやAPIエラーを発生させた際、シミュレーションが中断せずフォールバックメッセージを表示して継続すること。
- [ ] **オートポイエーシス証明ステータスの検証**: シミュレーションが進行し、すべての非メタシステムが一巡して条件を満たした時点で、画面上に「AUTOPOIESIS PROVEN」と動的に表示されること。また、初心者向け解説パネルのテキストが平易な日本語で破綻なく表示されていること。
- [ ] **攻殻風サイバーアバターの検証**: `AgentAvatar.tsx` が幾何学模様からネオン調の人型シルエットに変更され、発話時に周波数波形アニメーションが動作すること。
- [ ] **ターン制御の検証**: `fixed` および `dynamic` の両モードにおいて、1サイクル内に全非メタエージェントが漏れなく1回ずつ発言し、発言回数に公平性が保たれていること。
- [ ] Docker 環境 (`docker compose up`) で上記が検証可能であること。
- [ ] `./.shared-agents/harness/verify.sh` によるすべての静的チェック・ユニットテスト（eslint, tsc, ruff, mypy, pytest, Vitest）が正常にパスすること。

## 未解決・考慮事項
- **OP-1: 自己参照的連鎖グラフの動的更新タイミング**
  - オートポイエーシス証明の「自己参照ループ」の検出は、サイクル完了時のみ行うのか、あるいは毎ターンのエッジ更新時にグラフアルゴリズム（強連結成分の検出など）を用いてリアルタイム判定するのか。
- **OP-2: メタエージェント（モデレータ）のターン関与**
  - メタエージェント（`is_meta: true`）は発話者としてのターン数制限やターン一巡のルールからは完全に除外され、これまで通り議論の進行管理や選択のみを担当する設計で統一する。
