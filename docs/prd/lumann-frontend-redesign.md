# PRD: フロントエンド再デザイン（攻殻機動隊風・オートポイエーシス可視化）

## 概要と目標

本機能は、Luhmann Autopoiesis Simulation のフロントエンドを、現在の最小限の
チャットリスト UI から **攻殻機動隊風サイバーパンク世界観** を持つ
没入型ダッシュボードへと再デザインする。

**目標:**

1. **世界観の表現** — 攻殻機動隊を想起させるダーク・ネオン・グリッチ・HUD 風 UI により、
   ルーマンの社会システム理論を SF 的視覚言語で体感できるようにする。
2. **AI エージェントの人間化** — 各機能システム（Wissenschaft / Recht / Politik /
   Wirtschaft / Erziehung / Religion / Kunst …）を固有のアバターで表現し、
   発言時にパルス・思考中インジケータ等のアニメーションで「人間が話し合っている」
   感覚を演出する。
3. **オートポイエーシスのグラフィカル可視化** — エージェントをノード、発言を
   エッジとするネットワーク図により、発言が次の発言を連鎖的に呼び起こす
   「自己産出」のプロセスを視覚化する。
4. **リアルタイム社会構成の体感** — シミュレーション進行に伴い、ネットワークが
   成長し、エッジが太くなり、ノードが活性化していく様子を WebSocket / SSE
   ストリーム経由でリアルタイムに提示する。
5. **バックエンド互換性維持** — 既存 REST (`POST /api/simulations`,
   `GET /api/simulations/{id}/logs`) および WebSocket (`/ws/simulations/{id}`) /
   SSE (`/api/simulations/{id}/stream`) をそのまま活用し、バックエンド API の
   破壊的変更は行わない（YAGNI）。

非目標: 3D 化（Three.js）、音声合成、ユーザー認証、バックエンド大幅改修。

## ターゲットユーザー / ユースケース

### ターゲットユーザー
- **研究者 / 学生** — ルーマンの社会システム理論およびオートポイエーシス概念を
  直感的に理解したい社会学・哲学系の学習者。
- **デモ閲覧者** — LLM 多 エージェント シミュレーションの挙動を非技術者へ
  見せるためのショーケース観客。
- **開発者（自分自身）** — 本サンドボックスのエージェント挙動を視覚的に
  デバッグ・吟味したい作者。

### 主ユースケース
1. **UC-1 シミュレーション起動と観察**: ユーザーが「お題」と「最大ターン数」を
   入力して開始すると、攻殻風 HUD 上に各エージェントのアバターが配置され、
   発言が順次吹き出しで表示される。同時にネットワーク図が発言連鎖とともに
   成長していく。
2. **UC-2 過去シミュレーションの再読**: 完了した simulation_id を指定して
   ログ (`GET /logs`) を取得し、アニメーション付きで再生表示する。
3. **UC-3 統計のリアルタイム確認**: ターン数・各エージェント発言回数・平均発言長
   などの統計がシミュレーション中に更新される。
4. **UC-4 タイムラインの探索**: 時系列タイムラインをスクロール・ズームし、
   過去の任意の発言にジャンプできる。

## 機能要件 (必須)

### FR-1 攻殻機動隊風ワールドテーマ
- ダーク背景（#05060A 系）、ネオンブルー (#00E5FF) / ネーングリーン (#39FF14) /
  警告赤 (#FF2A6D) のアクセントカラー。
- 等幅フォント（`JetBrains Mono` または `Saira Mono`）を全体に適用。
- HUD 風フレーム・角枠・スキャンライン・グリッチエフェクトの CSS 装飾。
- Tailwind config に上記カラーパレットとフォントファミリを追加。
- `index.css` にスキャンライン / グリッチ / ネオン グロー のキーフレーム定義。

### FR-2 AI アバター表現
- 各エージェントを `agent_code`（二値コード）と `concern`（関心）から生成した
  **固有の SVG アバター** で表現（同一エージェントは常に同じアバター）。
- アバター生成は純関数 `generateAvatar(agentCode: string, concern: string): string`
  で SVG 文字列を返す。ハッシュベースの幾何学模様 + 関心キーワードのグリフ。
- アバターは円形枠内に表示、枠線色はエージェント固有色。
- **状態表現**:
  - `idle`（待機）: 静止
  - `thinking`（思考中）: 枠のパルスアニメーション + 内部スピナ
  - `speaking`（発言中）: ネオングロー強 + マイクアイコン
- 状態は現在発言中の `agent_name` と次の予想エージェント（fixed order の場合）から
  推論する。dynamic order の場合は thinking 状態を全員維持し、発言開始で切替。

### FR-3 発言吹き出し（チャットバルーン）
- 各発言 `Message` をアバター横の吹き出しとして表示。
- 発言者は左側、他エージェントからの応答はアバター位置に紐付いて表示。
- 吹き出しには `turn`, `agent_name`, `agent_code`, `provider/model` のメタ情報を
  小型 HUD ラベルで表示。
- 出現時にフェード + スライドイン アニメーション（framer-motion 採用可）。
- SSE ストリーム（`/stream`）の token ごとの表示にも対応可能にする（FR-7 と連動）。

### FR-4 オートポイエーシス ネットワーク可視化
- SVG または Canvas による **ノード（エージェント）+ エッジ（発言）ネットワーク図**。
- ノードは円、エッジは発言元 → 発言先（次ターンのエージェント）の矢印。
- 発言が追加されるたびにエッジがアニメーションで描画され、既存エッジは
  同ペア間の発言回数に応じて太く / 明るく変化する。
- ノードの大きさ / 輝度はそのエージェントの累積発言回数に比例して成長する。
- レイアウトは初期実装では 円形配置（agent_order 順）を維持（YAGNI）。
  力学レイアウト（d3-force 等）は次フェーズで検討。
- ネットワーク図はメインビューの中央に大きく配置し、吹き出しリストと並列表示。

### FR-5 リアルタイム社会構成の成長表現
- シミュレーション実行中、以下が WebSocket / SSE メッセージ到着ごとに更新される:
  - ノードの発言回数カウント（増加でサイズ + 輝度アップ）
  - エッジ本数 / 太さ（同ペアの発言回数）
  - 全体の「社会密度」メータ（エッジ数 / 完全グラフ比）
- 成長アニメーションは framer-motion の spring 系で自然に（急激なジャンプ防止）。

### FR-6 タイムライン
- 発言の時系列タイムラインを下部または右側パネルに表示。
- 各発言をタイムスタンプ付きのドット（turn 番号ラベル付き）でプロット。
- クリックでその発言の吹き出し位置へスクロール。
- 横スクロール可能（ターン数増加に追従）。
- ズーム操作は本フェーズでは最小限（2 段階切替）とし、詳細ズームは次フェーズ。

### FR-7 リアルタイム統計パネル
- 現在ターン数 / 最大ターン数
- 各エージェント発言回数（棒グラフまたは HUD バー）
- 平均発言文字数
- 経過時間（開始 timestamp からの差分）
- 使用プロバイダ / モデルの一覧
- これらは `messages` 配列から純関数で導出し、`messages` 更新ごとに再計算。

### FR-8 レイアウト構成
- 3 領域レイアウト（デスクトップ）:
  - 左: アバターリスト（エージェント一覧 + 現在の状態）
  - 中央: ネットワーク可視化（FR-4）+ 吹き出しリスト（FR-3）
  - 右: 統計パネル（FR-7）+ タイムライン（FR-6）
- モバイル / 狭画面では縦並びにスタック（Tailwind レスポンシブ）。

### FR-9 シミュレーションフォーム
- 既存 `SimulationForm` を HUD 風に再スタイル（機能変更なし）。
- `trigger_message`, `max_turns` 入力を維持。
- 開始ボタンはネオンアクセント。
- 実行中はフォーム無効化、ステータスを HUD インジケータで表示
  （idle / running / completed / failed）。
- オプション: `simulation_id` 入力欄を追加し、過去ログの再読み込みを可能にする
  （`GET /api/simulations/{id}/logs` を使用）。

### FR-10 エラー / 完了表示
- `failed` イベント時、HUD 風エラーバナーを表示（赤グリッチ）。
- `completed` 時、ネットワーク図が完成状態になり「SIMULATION COMPLETE」表示。

## 非機能要件 (パフォーマンス、UX等)

### NFR-1 パフォーマンス
- 100 ターン / 50 ノード相当まで DOM/SVG 再描画が 60fps を維持。
- ネットワーク図は SVG の場合 `React.memo` で差分再描画を抑制、
  または Canvas 実装で大規模対応（初期は SVG）。
- アバター SVG はメモ化（`useMemo`）し、`agentCode` 不変なら再生成しない。
- 統計計算は `messages` 配列の O(n) で済む範囲に抑える（n=数百程度）。

### NFR-2 型安全
- TypeScript strict モード、`any` 禁止。
- 既存 `Message` / `SimulationStartResponse` 型を再利用し、新規 UI 状態型は
  `types.ts` に Pydantic 的厳格さで定義。
- アバター生成関数は `string` 入力 / `string`（SVG）出力で明示型。

### NFR-3 アクセシビリティ
- ダークテーマでもコントラスト比 WCAG AA を満たす配色。
- アニメーションは `prefers-reduced-motion` で無効化可能。
- キーボード操作: フォーム入力 → Enter で送信、タイムライン ドットは
  Tab フォーカス可能。

### NFR-4 国際化
- 日本語ラベルを基本（既存 UI に準拠）、技術用語は英語併記可。

### NFR-5 コーディング制約
- コメント禁止（GEMINI.md 指定）。
- YAGNI: 3D / 音声 / 認証 / 永続化は導入しない。
- 新規依存ライブラリは最小限。アニメーションは `framer-motion` 1 本まで。
  ネットワークレイアウトは自前 SVG/Canvas 実装（d3 は次フェーズ検討）。

### NFR-6 後方互換性
- バックエンド API（REST / WS / SSE）への追加要求は行わない。
- 既存エンドポイントのレスポンス形状に依存する実装とする。

## 未解決・考慮事項

1. **アバター生成のリッチ化** — 現仕様はハッシュベース SVG。今後、プロキシ
   アバター生成 API（DiceBear 等）の導入余地あり。YAGNI 観点で次フェーズ判断。
2. **ネットワークレイアウト** — 円形固定は agent_order が `dynamic` の場合に
   順序が変わるとうまく表現できない可能性。力学レイアウト（d3-force）導入の
   要否を初期リリース後のユーザ反応で判断する。
3. **SSE vs WebSocket** — 既存 WS は発言全体を1メッセージで送信、SSE は
   token 単位ストリーミングを提供。FR-3 で「タイピング風」表示を本フェーズで
   入れるか次フェーズに回すか要判断（初期は WS ベースの完成メッセージ表示
   を主軸とし、SSE は AC には含めない）。
4. **依存ライブラリ選定** — `framer-motion` を唯一の追加依存として想定するが、
   純 CSS アニメーションで足りる範囲を最大化し、`framer-motion` を最小利用に
   する方針を技術仕様で確定する。
5. **エージェント状態推論** — 現状バックエンドは「次の発言者」を事前に通知
   しない。`thinking` 状態を適切に表示するには、`agent_order_mode=fixed` の
   場合は `agent_order` から次ターン推論可能。`dynamic` の場合は全員
   `thinking` 扱いとする擬似表示で妥協するか、バックエンドに `next_agent`
   イベント追加を要するかは Tech Spec で決定（本 PRD ではバックエンド拡張を
   スコープ外とする）。
6. **過去ログ再生速度** — AC-20 の過去ログ再生をリアルタイム相当の速度で
   アニメーションするか、即時一括表示するかは UX 調整余地あり。
7. **モバイル最適化の深さ** — 3領域レイアウトのモバイル版を縦スタックで
   済ますか、タブ切替 UI にするかはプロトタイプ評価後に決定。
